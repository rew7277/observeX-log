Endpoint

POST /api/v1/logs/ingest

Accepts raw strings, structured events, or arrays. Returns session id, ingestion count, processing time, detected apps, errors, warnings and health score.

Success

200 OK

{ "status": "success", "session_id": 123, "ingested": 18574, "processingTimeMs": 420, "stored": true }
Failures

Common responses

401 Missing or invalid token
400 Invalid payload
413 Payload too large
500 Internal server error

Search

GET /api/v1/logs/search

Query masked persisted logs using simple syntax like env=SANDBOX app=demo-checkout-api level=ERROR.

Trace

GET /api/v1/trace/<id>

Returns a grouped event timeline, surrounding context, failures, latency clues and supporting evidence rows.

Security

Safe by default

Docs use dummy keys only. Real keys are visible only inside authenticated workspace settings and should be rotated regularly.
